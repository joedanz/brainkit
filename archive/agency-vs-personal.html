Decision Brief · Agent Ownership Architecture

Which agents belong to the agency, and which to the advisor?

The split is not a fixed property of a capability — it flips with the employment contract. The right model defaults everything to personal and makes the agency earn each act of sharing, enforced structurally at the token layer, never by trusting the agent to police itself.

01The reframe

Contract dictates architecture — not the reverse.

The instinct is to label each capability "agency" or "personal" once and for all. That's the core error. The correct unit of analysis is capability × archetype: the same dossier is agency-owned for a W-2 employee and advisor-owned for a 1099 independent contractor.

1 · Mirror the contract

W-2 employees → the firm owns the book and dossier. 1099 ICs (the dominant luxury model) → the dossier and entire personal pipeline default to advisor-owned, segregated, host-invisible, and one-click exportable.

2 · Split, don't lump

Every "shared" asset decomposes into an institutional core that may pool (negotiated rates, reliability scores, compliance rules, booking records) and a personal layer that must not (voice, rapport, referral graph, advisor-brought suppliers).

3 · Earn sharing, structurally

Centralization must never silently expropriate the advisor. Fact-promotion is opt-in and hard-gated; portability is a shipped product feature; every boundary is enforced at the ZeroID/MCP layer — because agent self-policing runs ~112/129.

* Why zero are "purely personal": on a shared Hermes stack every capability rides shared infrastructure, so pure-personal assets — the advisor's voice model and referral graph — exist as components inside hybrids, not as standalone capabilities. They are first-class and portable, but they are not separate agents.
