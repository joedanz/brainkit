# Impeccable Design Context — Company Brain

## Design Context

### Users
Employee-first, dual-audience. Every employee opens the live dashboard (`brain dashboard`)
throughout the workday, alongside Obsidian and their AI Chief of Staff agent, to see the
state of their own compiled vault: what's new, what links where, what needs attention —
and to search, read, and capture notes. Company admins get the same UI with extra tabs
(People, Permissions, Promotions, Doctor) under the master lens; the admin experience is
secondary and must never make the personal experience feel like an ops console.

### Brand Personality
**Calm, trustworthy, literate.** A quiet reading room for your second brain. The product's
core promise is structural privacy — the interface should feel like it keeps that promise:
restrained, unhurried, typographic, warm. Confidence comes from composure, not density.
Nothing flashy, nothing that reads as "AI-generated dashboard."

Three words: *composed, literate, warm-mechanical* (plain markdown + git underneath — the
craft of good tools, not the gloss of SaaS).

### Aesthetic Direction
- **Editorial/library, not observability console.** Typography-led hierarchy; the knowledge
  graph is an artifact to study, not a neon centerpiece.
- **Both themes via a toggle**, `light-dark()` / OKLCH tokens. Dark is a dim reading room
  (warm-dark, never blue-black); light is good paper (tinted, never #fff). Respect
  `prefers-color-scheme` as the default, persist the user's explicit choice.
- Neutrals tinted toward the brand hue; one restrained accent used rarely (60-30-10).
- Distinctive display face paired with a refined body face (no reflex fonts — no Inter,
  Space Grotesk, Syne, IBM Plex, Fraunces, etc.). Fonts must be vendored — the dashboard
  is offline-first, CSP `default-src 'self'`, zero CDN.
- Anti-references: Grafana/Datadog density, crypto-dark neon, cyan-on-black "AI slop",
  gradient text, side-stripe cards, hero-metric tiles.

### Design Principles
1. **Privacy made visible.** The lens you're looking through (your vault vs. master) should
   always be quietly legible; write paths (capture → Inbox, promote → human gate) must show
   exactly where content goes and who will see it.
2. **Reading first.** Notes are the product. Rendered, readable notes beat raw `<pre>` dumps;
   line lengths capped ~70ch; generous leading on dark.
3. **Live means gentle.** Realtime updates arrive without yanking the view — settle, pulse
   softly, never reflow what the user is reading.
4. **Restraint over density.** Fewer, larger, better-set numbers; whitespace rhythm over
   card grids; the accent color is rare enough to mean something.
5. **Self-contained craft.** No build step, no CDN, textContent-only DOM. Every asset ships
   in the wheel. Constraints are part of the brand.

### Technical Constraints
- Strict CSP (`default-src 'self'`), no inline scripts/styles, no runtime network beyond `/api` + WS.
- Vanilla ES modules, vendored d3 + three.js; assets served by aiohttp from the package.
- Server binds 127.0.0.1; all writes must route through structurally safe primitives
  (`brain ingest` for capture, promotion queue for sharing) — never raw file writes.
